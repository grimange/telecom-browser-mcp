# Contract Source Of Truth Map

## Priority Mapping (as implemented)
1. Canonical contract models: `src/telecom_browser_mcp/contracts/m1_contracts.py`
2. Tool registration signatures: `src/telecom_browser_mcp/server/app.py`
3. Exported schemas: `docs/contracts/m1/*.schema.json` generated by `scripts/generate_contract_schemas.py`
4. Runtime validators: `src/telecom_browser_mcp/tools/service.py` using Pydantic model validation
5. Tests: `tests/contract/*`, integration/e2e tests
6. Docs: `README.md`, pipeline docs

## Observed Alignment
- M1 tools are in canonical map and schemas are generated from canonical models.
- Runtime validators use explicit models per tool.

## Drift Risk Spots
- `health` and `capabilities` are registered MCP tools but outside canonical `M1_TOOL_INPUT_MODELS` and outside canonical `ToolResponse` envelope.
