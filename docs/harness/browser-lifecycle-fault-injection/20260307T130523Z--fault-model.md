# Fault Model (20260307T130523Z)

Implemented deterministic lifecycle fault classes:

1. A Browser unavailable
- Trigger: `before_session_reuse`
- Action: `close_browser`
- Expected: classify as `BROWSER_SESSION_BROKEN`, invalidate stale session.

2. B Context invalidation
- Trigger: `after_context_created` or `before_wait_for_selector`
- Action: `close_context`
- Expected: bounded failure, cleanup reflected in result.

3. C Page invalidation
- Trigger: `before_click`
- Action: `close_page`
- Expected: normalized action failure (`UI_SELECTOR_FAILURE`) with retryable recovery.

4. D Stale selector
- Trigger: `after_selector_resolved`
- Action: `stale_selector` (DOM epoch bump)
- Expected: normalized stale-handle failure.

5. E Session registry inconsistency
- Trigger: modeled by stale browser/session reuse and explicit registry invalidation helper.
- Expected: stale handles removed, healthy sessions remain unaffected.

6. F Recovery logic errors
- Covered through assertions on classification, cleanup, retryability, and isolation.
