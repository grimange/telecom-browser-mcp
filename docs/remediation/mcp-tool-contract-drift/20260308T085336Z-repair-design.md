# MCP Tool Contract Repair Design

- timestamp: 2026-03-08T08:53:36Z

## Design Decisions

1. Canonical registration uses real orchestrator handlers.
- `server.tool(name=tool_name)(handler)` where `handler = getattr(app.orchestrator, tool_name)`.
- eliminates synthetic wrapper surfaces from schema inference.

2. Enforce strict argument models globally for MCP tools.
- set `ArgModelBase.model_config.extra = "forbid"` before tool registration.
- ensures published input schemas include `additionalProperties: false`.
- ensures invalid envelopes (for example `{"kwargs": ...}`) are rejected by validation.

3. Preserve dispatcher compatibility path only for non-MCP internal calls.
- `TelecomBrowserApp.dispatch` legacy normalization remains available for direct app dispatch compatibility.
- public MCP contract no longer depends on the legacy `kwargs` envelope.

## Rejected Alternatives

- keeping `**kwargs` wrappers and relying on `__signature__` mutation: rejected due drift susceptibility.
- broad acceptance of arbitrary `kwargs`: rejected because it hides contract defects.
